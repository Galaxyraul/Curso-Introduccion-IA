BLOQUE 01

Fundamentos de la IA

Definiciones esenciales, jerarquías de abstracción y el cambio de paradigma hacia el aprendizaje automático.

IA vs ML vs DL

  • IA (Inteligencia Artificial): Capacidad de una máquina para imitar el comportamiento humano inteligente a través de algoritmos.
  • ML (Machine Learning): Un subconjunto de la IA que permite a los sistemas aprender y mejorar automáticamente a partir de la experiencia.
  • DL (Deep Learning): Una evolución del ML basada en redes neuronales profundas que imitan la estructura del cerebro humano.

Paradigmas de Aprendizaje

Supervisado

Aprendizaje basado en datos etiquetados. El sistema conoce la respuesta correcta de antemano.

No Supervisado

Detección de patrones en datos sin etiquetas. El sistema agrupa información por similitudes.

Semi-supervisado

Híbrido que utiliza una pequeña cantidad de datos etiquetados con un volumen masivo sin etiquetar.

Por Refuerzo

Aprendizaje mediante interacción, maximizando recompensas a través de prueba y error.

BLOQUE 02

Preprocesamiento

La importancia de los datos: limpieza, gestión de outliers, normalización y preparación para el modelado.

Ley GIGO y tipos de datos

Garbage In, Garbage Out (GIGO)

Si alimentas al modelo con datos ruidosos o sesgados, el modelo aprenderá ruido y sesgo con absoluta confianza matemática. El algoritmo es simplemente un espejo.

Datos Excelentes + Modelo Simple ≫ Datos Mediocres + Modelo Complejo

Datos Tabulares

Estructurados en filas y columnas (SQL, Excel, CSV).

  • Dominio del ML Clásico.
  • Features claras (Edad, Salario).
  • Fácil interpretación matemática.

No Estructurados

Sin formato de tabla predefinido. Imágenes, Audio, Texto libre.

  • Requieren Deep Learning.
  • Alta dimensionalidad (píxeles).
  • Necesitan extracción de patrones compleja.

Cuidado con el Sesgo: El modelo no es machista ni prejuicioso; los datos históricos lo son.

Visualización y Análisis Exploratorio (EDA)

Histograma & KDE

Análisis Univariante: Permite ver la forma de la distribución (Gaussiana) y detectar sesgos (Skewness).

Scatter Plot

Análisis Bivariante: Detecta visualmente si la relación entre variables es lineal o si existen clusters definidos.

Box Plot (Caja/Bigotes)

El detector de mentiras. Resume la distribución y visualiza Outliers matemáticos.

IQR: [Q1 - 1.5·IQR, Q3 + 1.5·IQR]

Heatmap (Correlación)

Mide fuerza de relación (-1 a +1). Pearson (Lineal) y Spearman (Monótona). ¡Cuidado con Multicolinealidad!

"No te fíes de los números: el Cuarteto de Anscombe demuestra que hay que pintar los datos."

Estadística y Escalamiento

Dispersión

σ² = Σ(xi - μ)² / N

Varianza (σ²)

Mide qué tan lejos están los datos de la media. Es el promedio de distancias al cuadrado.

Desviación Típica (σ)

Recupera las unidades originales. Es la raíz cuadrada de la varianza σ².

Estandarización

z = (x - μ) / σ

Standard Scaler

Forzamos Media = 0 y Desviación = 1 en todas las columnas.

Z-Score

Pone a todas las features en la misma regla de medir, típicamente entre -3 y +3.

Tratamiento de Datos: Encoding e Imputación

Variables Categóricas

Rojo → [1, 0, 0]

One-Hot Encoding

Crea una columna por categoría. Ideal si no hay orden jerárquico.

Label Encoding

Asigna un número (0, 1, 2). ¡Cuidado! El modelo puede interpretar jerarquías falsas.

Valores Faltantes

Impute(X) = Mean | Median

Imputación

Rellenar con la Media o Moda para no perder instancias.

Estrategias

Eliminación de filas o uso de algoritmos avanzados para predecir el hueco.

Selección de Features

X_new ⊂ X_orig

Varianza Cero

Si una columna tiene siempre el mismo valor, no aporta nada. ¡Bórrala!

Relevancia

Prioriza columnas con relación matemática real con el objetivo (target).

Maldición de Dimensionalidad y PCA

La Maldición

Space ∝ Exp(Dims)

Espacio Disperso

A más dimensiones, los datos se alejan. La distancia euclidiana deja de tener sentido.

Consecuencia

Necesitas un volumen de datos exponencial para entrenar correctamente.

PCA (No Supervisado)

Max(Variance)

Estrategia

Simplifica proyectando en menos dimensiones preservando la estructura global.

Analogía

Como un fotógrafo buscando el ángulo donde el grupo de personas se vea mejor (máxima dispersión).

LDA: Reducción Supervisada

LDA (Supervisado)

Max(Separability)

Estrategia

Utiliza las etiquetas (y) para encontrar el eje que mejor separa las clases.

Diferencia con PCA

A LDA le importa qué hace diferentes a las clases, no la forma global de la nube.

Criterio de Éxito

Between / Within

Entre-Clases

Maximizar la distancia entre los centros de los diferentes grupos.

Intra-Clase

Minimizar la dispersión interna: que los puntos de una clase estén muy juntos.

BLOQUE 03

Arquitectura de un algoritmo

Desmontando la caja negra: Representación (El Cuerpo), Evaluación (El Juez) y Optimización (El Motor).

Anatomía de un Algoritmo de ML

Representación

Espacio de Hipótesis

El "Cuerpo"

¿Qué es capaz de aprender? Es el conjunto de funciones que el algoritmo puede utilizar.

Límite Físico

Si eliges una línear recta, el modelo jamás entenderá un círculo. Ej: Árboles, Grafos, Hiperplanos.

Evaluación

Función de Coste J(w)

El "Juez"

¿Cómo distinguimos un modelo bueno de uno malo? Mide el error en un solo número (escalar).

Objetivo

Minimizar la discrepancia entre la predicción y la realidad. Ej: MSE, Log Loss.

Optimización

Búsqueda de w*

El "Motor"

¿Cómo buscamos la mejor configuración? Ajusta los pesos iterativamente para ganar al juez.

Método

Navega por el espacio de parámetros. Ej: Descenso del Gradiente, Programación Cuadrática.

El Juez: La Función de Coste (Loss)

MSE (Mean Squared Error)

Σ(y - ŷ)² / N

Castigo al Outlier

Al elevar al cuadrado, un error grande penaliza muchísimo más que uno pequeño. Induce a modelos conservadores.

MAE (Mean Absolute Error)

Σ|y - ŷ| / N

Robustez

No entra en pánico con valores atípicos. Trata todos los errores de forma lineal.

Log Loss (Cross-Entropy)

- Σ [y·log(ŷ) + (1-y)·log(1-ŷ)]

Confianza Equivocada

Penaliza la "arrogancia". Si el modelo está muy seguro de algo falso, el coste tiende a infinito.

El Motor: Estrategias de Descenso

Batch Descent

El Prudente

Lee todo el dataset para dar un solo paso. Gradiente perfecto, pero lento e inviable en Big Data.

Stochastic (SGD)

El Borracho

Toma un solo dato al azar y da un paso inmediato. Muy rápido pero ruidoso y errático.

Mini-Batch

El Estándar

Usa lotes (32, 64, 128). Equilibra velocidad y estabilidad. Optimizado para el hardware (GPU).

w_nuevo = w_viejo - α · ∇J(w)

Evolución Moderna: Optimizador Adam

Momentum (Inercia)

Acumula velocidad de gradientes pasados. Permite saltar baches pequeños (mínimos locales) para encontrar el valle profundo real.

Adaptive Learning Rate

Ajusta el paso (α) individualmente para cada peso. Si una variable es rara, le da pasos de gigante; si es inestable, la frena.

Adam es el "estándar de oro" por ser robusto y converger mucho más rápido sin ajustes manuales constantes.

Regularización: El Freno al Overfitting

Coste_total = Error(MSE) + λ · Penalty(w)

L1 (Lasso)

Fuerza los pesos inútiles a ser exactamente cero. Realiza selección automática de variables.

L2 (Ridge)

Fuerza los pesos a ser pequeños pero no cero. Evita que una sola variable domine toda la predicción.

"Tu trabajo es minimizar el error, pero te voy a multar por cada peso alto que uses". El modelo busca la solución más simple que explique bien los datos.

BLOQUE 04

Modelos Clásicos

Explorando la "caja de herramientas" esencial: Regresiones, SVM y Modelos Bayesianos.

Regresión Lineal: El Hiperplano

Representación

ŷ = wᵀx + b

Aproximación Lineal

Predice un valor continuo (escalar) como suma ponderada de las entradas.

Evaluación

Min Σ(y - ŷ)²

MSE (Mínimos Cuadrados)

El "Juez" penaliza cuadráticamente los residuos para encontrar la línea de mejor ajuste.

Analogía

Mesa de Mezclas

El entrenamiento ajusta los faders (pesos w) de cada canal (feature) para minimizar la distorsión final.

Profundidad: Al ser una función cuadrática y convexa, garantizamos encontrar siempre el mínimo global mediante Descenso del Gradiente.

Regresión Logística: Probabilidades

Función Sigmoide

σ(z) = 1 / (1 + e⁻ᶻ)

Aplastamiento

Mapea cualquier número de (-∞, +∞) al rango [0, 1]. Transforma rectas en probabilidades.

Clasificación

P(y=1|x) > 0.5

Frontera de Decisión

Define un hiperplano que separa dos clases. A pesar del nombre, es un algoritmo de clasificación.

Evaluación

Log Loss

Entropía Cruzada

Penaliza logarítmicamente la confianza errónea ("arrogancia" del modelo).

SVM: Geometría de Margen Máximo

La Calle más Ancha

Max (2 / ||w||)

Margen Máximo

Busca separar las clases con la mayor distancia posible, minimizando el riesgo de error.

Kernel Trick

Φ(x)ᵀΦ(x')

No Linealidad

Proyecta datos a dimensiones superiores para separar nubes que en 2D parecen imposibles.

Vectores Soporte

Puntos Críticos

El modelo solo depende de los puntos más cercanos a la frontera. Son los pilares del hiperplano.

Naive Bayes: Causalidad Inversa

Teorema de Bayes

P(C|x) ∝ P(x|C)P(C)

Probabilidad Posterior

Calcula la probabilidad de una clase basándose en las evidencias (features) observadas.

El supuesto "Naive"

Independencia

Asume que los rasgos no tienen relación entre sí (ej: el color no depende de la forma). ¡Es falso, pero funciona!

Dominio NLP

Texto y Spam

Líder indiscutible en velocidad y clasificación de texto gracias a su simplicidad matemática.

Caja de Herramientas: ¿Cuál elegir?

Simplicidad Lineal

L. Reg / Log. Reg: Rápidos, interpretables y base de toda la estadística moderna. Úsalos como primer baseline.

Potencia Geométrica

SVM: Ideal para datasets pequeños pero complejos donde la frontera de decisión no es una recta.

Velocidad Probabilística

Naive Bayes: Tu mejor opción para filtrado de texto en tiempo real y Big Data ligero.

Regla de Oro

"La complejidad del modelo debe ser proporcional a la cantidad de datos". Evita usar un Ferrari (Deep Learning) para ir a la esquina (100 filas).

BLOQUE 05

Entrenamiento y Evaluación

¿Cómo sabemos si el modelo funciona? Validación Hold-out, Cross-Validation, Matrices de Confusión y el equilibrio del F1-Score.

El Equilibrio: Bias vs Variance

Underfitting (Perezoso)

Alto Bias (Sesgo)

Síntoma

Mucho error tanto en Entrenamiento como en Test. El modelo es demasiado simple.

Analogía

Estudiante que solo lee la contraportada del libro. No aprueba ni los ejercicios de clase.

Overfitting (Memoria)

Alta Varianza

Síntoma

Acierto del 99% en Entrenamiento, pero 50% en Test. El modelo memoriza el ruido.

Analogía

Estudiante que memoriza las respuestas exactas (A, B, C...) de exámenes viejos sin entender.

Protocolo de División: Las 3 Fases

Training (70%)

El Libro de Texto

Datos para que el algoritmo aprenda los patrones y ajuste sus pesos (w).

Validation (15%)

Simulacro de Examen

Se usa para ajustar los Hiperparámetros (k, C, Learning Rate) sin "quemar" el test.

Test (15%)

Examen Oficial

Se usa UNA SOLA VEZ al final. Es la nota definitiva para el cliente.

Random State: Antes de dividir, baraja. Fijar una semilla (ej. 42) asegura que tus experimentos sean reproducibles.

K-Fold Cross-Validation: El Estándar

Estrategia de Rotación

Dividimos el dataset en K partes. Entrenamos K veces, rotando cuál parte sirve de validación.

Promedio(Error₁, Error₂, ..., Errorₖ)

¿Cuándo usar cada una?

  • Hold-out: Muchos datos (>10k).
  • K-Fold: Datos medios (<10k).
  • LOOCV: Datos minúsculos (<100).

Métricas: La Matriz de Confusión

True Positive (TP)

Acierto: Predije Cáncer y TIENE Cáncer.

True Negative (TN)

Acierto: Predije Sano y ESTÁ Sano.

False Positive (FP)

Falsa Alarma: Predije Cáncer pero está sano.

False Negative (FN)

Peligro Mortal: Predije Sano pero tiene cáncer.

La Trampa del Accuracy: En detección de terroristas (99.9% honestos), un modelo que diga "Nadie es terrorista" sacará un 99.9% de acierto pero será inútil.

Precision vs Recall: El Dilema

Precision (Pura)

TP / (TP + FP)

¿De los que dije que eran positivos, cuántos lo eran de verdad?

Ideal para Filtros de Spam.

Recall (Sensibilidad)

TP / (TP + FN)

¿De todos los reales, cuántos fui capaz de encontrar?

Ideal para Diagnóstico Médico.

F1-Score

2 · (P · R) / (P + R)

Media armónica. Se usa para buscar un equilibrio cuando no tenemos preferencia clara.
